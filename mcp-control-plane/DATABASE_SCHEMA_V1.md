# Database Schema V1

Status: SQLite/libSQL canonical

The canonical schema is the goose migration set in `db/migrations/`.

## Storage Rules

- SQLite/libSQL is the only supported platform persistence layer.
- Migrations are managed by goose.
- Production SQL is generated by sqlc from `db/queries/`.
- Internal durable IDs use UUIDv7 and are stored as 16-byte SQLite `BLOB` values.
- JSON payloads and contracts are stored as `TEXT` with `json_valid(...)` checks.
- Timestamps are stored as UTC RFC3339 text values or SQLite `CURRENT_TIMESTAMP` defaults.
